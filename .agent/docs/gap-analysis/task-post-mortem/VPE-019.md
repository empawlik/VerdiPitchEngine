# Explorative Task Post-Mortem: VPE-019

> [!ABSTRACT] Core Concept
> Evaluates the failure of `ffmpeg` to preserve exact Roon grouping metadata (e.g., MusicBrainz IDs, Vorbis comments) during pitch conversion, and proposes a `metaflac` injection pipeline to ensure absolute metadata inheritance.

## 1. Current State Evaluation
Currently, the Verdi Pitch Engine leverages `ffmpeg` with `-map_metadata 0` to preserve FLAC metadata during the 432 Hz conversion process. However, this approach fails to achieve 1:1 metadata parity. `ffmpeg` strips critical non-standard Vorbis comments (such as `MusicBrainz Release Group Id`) and embedded artwork. As a result, when the newly processed files are scanned by Roon, they are incorrectly recognized as a completely separate, new album rather than being appropriately grouped as a "Version" of the existing album.

## 2. Pros and Cons
**Pros (of current `ffmpeg` metadata mapping):**
- Simple to implement entirely within the audio transcoding pipeline.
- Requires no additional dependencies.

**Cons:**
- Breaks strict 1:1 data integrity.
- Causes media library pollution by generating isolated album entities in Roon.
- Drops high-value MusicBrainz identification tags.

## 3. Alternative Solutions
- **ExifTool Mapping:** We could invoke ExifTool post-conversion to clone the metadata block. ExifTool is powerful but introduces a heavy dependency that is often slow and less native to FLAC than `metaflac`.
- **Go-native FLAC Tagging (`go-flac`):** Parsing and injecting Vorbis comments strictly via a Go library. This would remove shell dependencies but could be complex to implement bit-perfect preservation of embedded cover art (PICTURE blocks).

## 4. Brainstorming / Future Enhancements
1. **[VPE-020] Metaflac Injection Pipeline**: Implement a post-transcode step in `ProcessFile` that utilizes the native `metaflac` utility to perform absolute metadata inheritance. The pipeline will pipe the output of `--export-tags-to=-` from the original FLAC directly into `--import-tags-from=-` on the 432 Hz converted FLAC.
2. **[VPE-021] Cover Art Preservation**: Extend the `metaflac` extraction logic to also export the `--export-picture-to=` block and append it to the processed file, as standard tag importation does not cover embedded images.

## 5. ATLAS Documentation Usage
No new ATLAS documentation was injected for this analysis. The gap was identified through empirical Roon library behavior and verified against known `ffmpeg` FLAC metadata limitations.
