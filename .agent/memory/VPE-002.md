# Technical Memory: VPE-002

## E2E Testing with FFmpeg Dependencies

### Context
Testing the native `ProcessFile` invocation of the Verdi Pitch Engine requires an underlying installation of `ffmpeg` containing the `rubberband` filter (configured as `rubberband=pitch=432/440`). In decentralized and CI environments (such as a developer's local MacBook with default Homebrew FFmpeg), this library is often absent, causing the standard integration test to throw: `No such filter: 'rubberband'`.

### Resolution
To prevent CI/CD thrashing while ensuring integration testing remains possible in compatible environments (like NAS-hosted containers), we implemented a conditional environment check:

```go
func hasRubberband() bool {
	cmd := exec.Command("ffmpeg", "-filters")
	output, err := cmd.Output()
	if err != nil {
		return false
	}
	return bytes.Contains(output, []byte("rubberband"))
}
```

The E2E test gracefully runs `t.Skip("ffmpeg does not have rubberband filter compiled in")` if the host lacks the necessary binaries. This adheres to our graceful failure directives and zero-trust evaluation constraints.
