# Technical Resolution: VPE-013 - Roon Metadata In-Place Preservation

## Architectural Decision
The Verdi Pitch Engine output format was refactored to support metadata preservation for music server ecosystems like Roon. Instead of outputting the processed 432 Hz audio into a new `[432 Hz]` folder alongside the `[440 Hz]` original folder, the process has been updated to perform an in-place replacement. The original 440 Hz folder is moved to a hidden backup directory (`.Folder [440 Hz]`) and the 432 Hz files are output to the exact original directory path. 

## Implementation Details
- **In-Place Output:** The original album path is preserved for the 432 Hz output. The 440 Hz files are backed up in a hidden directory.
- **Hidden Backups:** The 440 Hz backup folder is hidden with a leading dot (e.g., `.` prefix) to prevent Roon from indexing duplicate tracks.
- **Timestamp Synchronization:** The original access and modification timestamps (`ModTime`, `AccessTime`) from the 440 Hz media files and directories are copied directly to the new 432 Hz output. This prevents Roon and other media scanners from treating the converted files as "Newly Added", preserving favorites, playlists, and exact import dates.
- **Sidecar Metadata:** Preserved the migration of sidecar metadata files (Art, Lyrics, PDFs) to ensure full album completeness alongside the timestamp retention.
