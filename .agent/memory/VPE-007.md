# VPE-007 Technical Memory

## Architectural Resolutions & Insights

### 1. Librubberband Integration
We transitioned from the native FFmpeg `asetrate` filter to the `rubberband` library. `asetrate` alters both pitch and tempo simultaneously, whereas `rubberband` employs Time-Scale Modification (TSM) to shift pitch to 432 Hz while strictly preserving the track's original duration and tempo, maintaining the acoustic integrity of the recording.

### 2. High-Resolution Audio Preservation (24-bit)
To prevent downsampling Audiophile-grade masters, we implemented an `ffprobe`-based stream detection phase. The engine now mathematically extracts the original bit-depth and natively handles 24-bit audio scaling inside the FFmpeg processing graph without defaulting back to 16-bit.

### 3. Container Station Pathing Orchestration
Operating directly on a QNAP NAS required an intermediate mapping logic. The `scripts/verdi-process.sh` wrapper automatically intercepts host-level paths (e.g., `/share/Multimedia/Audio/...`) and auto-corrects them into the internal Docker volume mount paradigm (`/music/...`).

### 4. CLI Flag vs. Environment Variable Precedence
We identified a defect where persistent environment variables baked into the `Dockerfile` (`VERDI_IN`, `VERDI_OUT`) were unconditionally overriding user-supplied CLI flags during `verdi-process.sh` execution. This was resolved by implementing `flag.Visit` in the Go entrypoint, ensuring that explicit CLI commands take absolute precedence over container environment variables.
