# Technical Resolution: VPE-010 - UI & Timeouts

## Architectural Decision
Batch processing of extremely large lossless files across a NAS via `ffmpeg` poses the risk of invisible thread locking if `ffmpeg` hangs during processing (e.g., due to edge-case corrupted tracks or `librubberband` processing failures). By enforcing `exec.CommandContext` with a 15-minute `context.WithTimeout` per track, the Golang orchestrator retains hard administrative control and cleanly prunes zombie processes.

## Implementation Details
- **Microsecond Precision:** `out_time_us` is extracted directly from the stdout pipe using a fast `bufio.Scanner`, converting `string` to `int64` via `strconv.ParseInt(usStr, 10, 64)`.
- **UI Containerization:** A standard 60-width terminal component handles the rendering context dynamically per active worker. Idempotency skips correctly increment the overall bar without creating zombie worker bars.
- **Coverage Rebounds:** Tested error fallback behavior in `getDuration` ensures 80%+ safety coverage for the `internal/converter` execution logic.
